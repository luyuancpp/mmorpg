# `run/` — Runtime Artifacts

This directory holds **transient runtime artifacts** generated by local
development: logs, PID files, and scratch dumps. It is separate from `bin/`
(C++ runtime working dir + build outputs) so that:

- `git status` stays clean (`run/` is fully gitignored).
- `rm -rf run/` is safe — it never deletes binaries, configs, or build state.
- CI/Docker stages copy `bin/` into images without dragging local logs along.

## Layout

```
run/
├── logs/
│   ├── go_services/          # Go service stdout/stderr (per service)
│   ├── cpp_nodes/            # C++ node stdout/stderr (per instance)
│   ├── robot/                # Robot load-test client logs
│   └── sa_token.log          # SA-Token (Java) dev server log
├── pids/
│   ├── go_services.pid.json  # Tracked PIDs for Go services
│   ├── cpp_nodes.pid.json    # Tracked PIDs for C++ nodes
│   └── sa_token.pid          # SA-Token PID
└── scratch/                  # Transient dumps: test durations, debug grep output, etc.
```

## Who writes here

- `tools/scripts/go_services.ps1`  → `run/logs/go_services/`, `run/pids/go_services.pid.json`
- `tools/scripts/cpp_nodes.ps1`    → `run/logs/cpp_nodes/`, `run/pids/cpp_nodes.pid.json`
- `dev.bat` (SA-Token start/stop)  → `run/logs/sa_token.log`, `run/pids/sa_token.pid`

See `docs/ops/log-management.md` and `bin/README.md` for the rationale.
