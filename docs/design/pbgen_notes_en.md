## Node Restart / Network Disconnect Safety (2026-03-14)

### UUID-based identity guarantees
- Every node startup generates a fresh UUID via `boost::uuids::random_generator`.
- ALL node comparisons MUST use `NodeUtils::IsSameNode(uuid1, uuid2)`, never raw `node_id`.
- **BUG / TODO**: `ConnectToGrpcNode` has no UUID check — needs the same UUID guard as the TCP path.

### Reconnect ≠ Data Refresh (critical architectural rule)
- **NEVER couple business data sync to raw TCP reconnect/disconnect events.**
- Correct pattern: TCP reconnect only rebuilds the transport layer. Data sync is driven by **explicit business messages** sent once the handshake succeeds.

## contracts/kafka C++ link pitfall

## pbgen event-id rollout notes
- Running `pbgen` currently also regenerates `tools/robot/proto/**` tracked `.pb.go` files as a side effect.
- `pbgen` can fail late if `tools/robot/logic/handler/` does not exist.
- Gate Kafka handler should dispatch each consumed `GateCommand` exactly once.

## Windows pbgen C++ gRPC generation
- Can silently fail when `--plugin=protoc-gen-grpc=grpc_cpp_plugin.exe` relies on PATH; resolve plugin with absolute path.
- Gate C++ gRPC generation by `domain_meta.<domain>.rpc.type` rather than directory-level proto service scans.
