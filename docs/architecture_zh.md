# 事件系统
---
## 1. 优先级
    异步事件处理用于需要解耦的场景——例如，当一个与副本相关的操作无法立即执行时。
    异步事件按优先级分类，在高负载下每帧只处理重要消息；不重要的消息可以丢弃。
    如果消息需要持久化，可以存储在玩家实体上，下次登录时处理。
    异步事件不得依赖顺序；如果需要严格顺序，使用同步（直接调用）事件。

## 2. 优先级分类
    0 — 关键：在 tick 结束时处理；未完成则持久化到数据库
    1 — 普通：负载下降后处理
    2 — 可丢弃：非必要消息（如 UI 提示通知）
