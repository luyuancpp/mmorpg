# Proto-Gen 命名迁移检查清单

本文档定义了从历史名称 `pbgen` 迁移到用户侧名称 `proto-gen` 和规范内部名称 `protogen` 后的当前命名边界。

## 当前策略

- 在用户侧文档、任务、启动配置和终端帮助中使用 `proto-gen`。
- 在规范的磁盘源码和模块名中使用 `protogen`：
  - `tools/proto_generator/protogen/`
  - `tools/proto/protogen/`
  - `tools/docs/protogen/`
- 主要用户侧二进制文件为 `proto-gen.exe`；保留 `pbgen.exe` 作为兼容副本。
- 在命令、二进制副本和历史环境变量中保留 `pbgen` 作为兼容别名。
- 将生成的头文件/注释（如 `Code generated by protogen`）视为生成器拥有的输出。

## 迁移目标

- 减少新贡献者的困惑。
- 在过渡期间保留现有本地脚本和 CI 流程。
- 避免部分重命名导致运行时行为与文档不一致。

## 安全的低风险变更

- 更新 README 文件和 AGENTS 指引，优先使用 `proto-gen`。
- 将 VS Code 任务标签和启动标签重命名为 `proto-gen`。
- 添加 `proto-gen-build` 和 `proto-gen-run` 别名，同时保留 `pbgen-build` 和 `pbgen-run`。
- 保持帮助输出和示例与用户侧名称一致。

## 中等风险变更

- 添加仅向新用户暴露 `proto-gen` 的包装脚本。
- 更新归档日志/自述文件中的措辞，路径仍为 `docs/protogen/`。

在发布中等风险变更之前，请验证：

- VS Code 任务在全新检出后仍可正常工作。
- `dev_tools.ps1 -Command proto-gen-build` 和 `proto-gen-run` 无需手动环境设置即可运行。
- 使用 `pbgen-build` / `pbgen-run` 的现有用户不受影响。
- 没有 CI 或本地自动化流程假定只存在 `pbgen.exe`。
- `PROTOGEN_ENABLE_PPROF=1` 正常工作，旧的 `PBGEN_ENABLE_PPROF=1` 行为不变。

## 高风险变更

- 移除兼容副本 `pbgen.exe`。
- 移除兼容别名 `pbgen-build` 和 `pbgen-run`。
- 停止接受 `PBGEN_ENABLE_PPROF`。
- 将 Go 模块/包/导入路径从 `protogen/...` 改为其他内部命名方案。
- 在未验证生成器差异的情况下更改生成输出横幅 `Code generated by protogen`。

在进行任何高风险重命名之前，请验证：

- 在 `tools/proto_generator/protogen/` 中 `go build ./...` 成功执行。
- 所有内部导入已一致更新。
- 启动配置、脚本和任务已一并更新。
- 所有生成文件的差异已确认为有意变更，而非无意的代码流转。
- 本地工具链所需的历史引用已被替换或明确保留。

## 推荐的发布顺序

1. 保持当前状态：用户侧 `proto-gen`，内部 `protogen`，兼容 `pbgen`。
2. 根据 `protogen` 目录布局验证脚本、任务和本地文档。
3. 在现有自动化仍依赖 `pbgen.exe` 期间，保留兼容包装器或重复输出。
4. 仅在此之后，才考虑在专门的验证过程中移除历史别名。

## 停止条件

如果出现以下任何情况，停止迁移并回退到上一个稳定的兼容层：

- CI 或本地生成脚本因假定硬编码的 `pbgen` 路径而失败。
- Go 导入在文件间出现分歧，产生部分重命名破坏。
- 生成输出在命名范围之外发生意外变化。
- 团队工作流仍依赖未经审计的历史路径。

## 剩余的有意保留的 `pbgen` 引用

以下 `pbgen` 引用是当前预期存在的，不应被视为遗漏的清理项：

- 兼容路径和目录：
  - `tools/proto_generator/protogen/`
  - `tools/proto/protogen/`
  - `tools/docs/protogen/`
- 二进制文件和环境变量名：
  - `proto-gen.exe`
  - `pbgen.exe`
  - `PROTOGEN_ENABLE_PPROF`
  - `PBGEN_ENABLE_PPROF`
- 兼容命令和帮助文本：
  - `pbgen-build`
  - `pbgen-run`
- 内部源码级标识符：
  - Go 模块/导入路径如 `protogen/...`
  - `tools/proto_generator/protogen/` 下的包级注释或日志路径
- 生成或归档内容：
  - `generated/` 下已提交的生成输出
  - `tools/docs/protogen/` 下的历史运行日志和提取的致命错误行

这些类别仅应在专门的兼容性迁移中变更，而非在日常文档清理中处理。

## 建议

目前，仓库处于正确状态：

- 用户侧名称：`proto-gen`
- 内部规范名称：`protogen`
- 兼容名称：`pbgen`

在有专门的迁移任务并分配了明确的验证时间之前，不要重命名磁盘路径或包导入。
