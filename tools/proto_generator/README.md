# Proto Generator (proto-gen)

Canonical protocol buffer code generation tool for the MMORPG project.

## Overview

proto-gen (historical name: pbgen) is a code generator that processes `.proto` files and generates language-specific bindings for:
- Go gRPC services
- C++ gRPC stubs
- Java protobuf classes
- Python protobuf definitions

## Directory Structure

```
proto_generator/
├── protogen/              # Canonical proto-gen source tree
│   ├── cmd/               # CLI entrypoint and pipeline bootstrap
│   ├── internal/          # Generator implementations and helpers
│   ├── etc/               # Configuration files
│   │   └── proto_gen.yaml # Default generation config
│   └── logger/            # Shared logger setup
└── README.md
```

## Configuration

The main configuration file is `protogen/etc/proto_gen.yaml`. It specifies:
- Input proto directories
- Output directories for each language
- Code generation options
- Plugin paths (e.g., gRPC C++ plugin)

## Usage

If you are not sure which command to use, start with:

```powershell
pwsh -File ../../scripts/dev_tools.ps1 -Command help
```

### Build proto-gen

```powershell
pwsh -File ../../scripts/dev_tools.ps1 -Command proto-gen-build
```

### Generate Code

```powershell
# Using default config
pwsh -File ../../scripts/dev_tools.ps1 -Command proto-gen-run

# Using custom config
pwsh -File ../../scripts/dev_tools.ps1 -Command proto-gen-run -ConfigPath etc/proto_gen.yaml
```

## Output

Generated code is placed in:
- `generated/proto/` - All language proto outputs
- `go/generated/proto/` - Go-specific bindings
- `cpp/generated/` - C++ protobuf and gRPC stubs
- `java/` - Java generated classes

## Logs

Generation logs are stored in:
- `protogen/` directory (*.log, *.txt files)
- Check `docs/protogen/` for historical run logs

## Naming Transition Policy

- Prefer `proto-gen` in user-facing docs, tasks, scripts, and conversation.
- Use `protogen` as the canonical internal directory and Go module name.
- Keep `pbgen-build` and `pbgen-run` as compatibility aliases in `dev_tools.ps1`.
- Build output now produces `proto-gen.exe` as the primary user-facing binary while still retaining `pbgen.exe` as a compatibility copy.
- Prefer `PROTOGEN_ENABLE_PPROF=1`; the old `PBGEN_ENABLE_PPROF=1` remains accepted for compatibility.
- Treat generated file headers like `Code generated by protogen` as generator-owned output, not a documentation cleanup target.

See also:

- `../docs/proto_gen_naming_audit.md` for the current naming-state snapshot.
- `../docs/proto_gen_naming_migration.md` for retained compatibility boundaries and future migration rules.

## Notes

- proto-gen is idempotent; running it multiple times produces the same output
- `dev_tools.ps1` still accepts legacy aliases `pbgen-build` and `pbgen-run`
- Always validate generated code before committing
- Keep config file in sync with actual proto locations
